## C12 Spike: GNURadio OOT Module + gr-iio for Pluto+ SDR

### Context
Shape C uses a GNURadio out-of-tree (OOT) module (`gr-lunalink`) to integrate the
Pluto+ SDR for the V6 demo. The module wraps `lunalink_core` in a GNURadio source block
(`afs_source`) that outputs composite AFS IQ as `gr_complex` samples. The gr-iio block
then drives the Pluto+ TX at 2492.028 MHz. On the RX side, a Pluto+ source block feeds
a log-power FFT block whose output is published via ZMQ to the web app's spectrum panel.

The original libiio-wrapper approach (C12-Q1 through C12-Q7) is superseded by this design.

### Goal
Determine the concrete steps to build a GNURadio OOT module that wraps `lunalink_core`,
confirm gr-iio achieves the required sample rate and frequency at 2492.028 MHz, and
establish how FFT magnitude data flows from the flowgraph to the web app.

### Questions

| # | Question |
|---|----------|
| C12-Q1 | What is the GNURadio OOT module structure generated by `gr_modtool`? Which files are boilerplate vs which contain our logic, and how does the CMake integrate with the existing `lunalink` build? |
| C12-Q2 | What is the correct GNURadio block type for `afs_source`? A `gr::sync_block` producing `gr_complex` at a fixed rate, or a `gr::block` with variable output length matched to one 12-second AFS frame at a time? |
| C12-Q3 | Does gr-iio support the ADALM-PLUTO+ (not the original ADALM-PLUTO) and can it tune to 2492.028 MHz? What is the actual frequency resolution of the AD9363 PLL at that frequency and is the offset acceptable for AFS reception? |
| C12-Q4 | What is the maximum IQ sample rate achievable via gr-iio on the Pluto+? Is 2× the AFS-Q chip rate (≥10.23 MSPS) achievable for a faithful composite signal, or are we limited to AFS-I only (≥2.046 MSPS)? |
| C12-Q5 | What is the standard GNURadio mechanism for forwarding FFT magnitude data to an external process (e.g., the FastAPI WebSocket server)? Is `gr::zeromq::pub_sink` the right block, and what is the typical latency at 1 Hz update rate? |
| C12-Q6 | What is the GNURadio installation story on macOS vs Linux for demo day? Is the Goonhilly demo machine likely Linux? Is there a conda/pip install path or does it require building from source? |
| C12-Q7 | Are there known reliability issues with gr-iio + Pluto+ at the required sample rate (USB underruns, buffer sizing, frequency drift) that require defensive design in the flowgraph? |

### Acceptance
Spike is complete when all questions are answered and we can describe: the OOT module
file structure and CMake integration, the correct block type and API for `afs_source`,
the achievable sample rate and frequency accuracy at 2492.028 MHz via gr-iio, the
ZMQ → WebSocket data path to the web app, the GNURadio install approach for demo day,
and any defensive measures needed for reliable operation.
