# LunaLink — AFS Reference Implementation — Frame

## Source

> This code is meant to fly on a satellite and be used for space mission.

> ESA is presenting a pilot programme on behalf of The Consultative Committee for Space Data Systems (CCSDS)... a competition to develop high-quality reference implementations of selected CCSDS protocols. [...] LunaNet Signal-In-Space Recommended Standard – Augmented Forward Signal (LSIS – AFS). The LSIS–AFS standard defines how lunar orbiters or surface systems broadcast a unified navigation and timing signal to support future missions on and around the Moon.

> Milestone 1 — Standards Compliance & Architecture design prepared: 31 March 2026
> Milestone 2 — Protocol Implementation complete: 31 July 2026
> Milestone 3 — Validation and Interoperability Demonstration complete: 31 August 2026

> Deliverables: LSIS–AFS compliance matrix, signal chain block diagram, implementation architecture description, software implementation of LSIS–AFS signal framing, modulation and coding per specification, test vectors and validation methodology, BER performance demonstration, cross-decoding of independently generated LSIS–AFS signals.

## Problem

The `lunalink` Python/C++ package exists as a hardened foundation (ESA coding guidelines, sanitizers, clang-tidy, 90%+ coverage gates) but contains only a placeholder `add()` function. The actual LSIS-AFS signal — a GNSS-like lunar navigation broadcast at 2492.028 MHz — has not been implemented. The competition requires a credible, standards-traceable reference implementation and an interoperability demonstration.

---

## Outcome

A working, documented LSIS-AFS reference implementation in the existing `lunalink` package that:
- generates correct AFS-I (data) and AFS-Q (pilot) baseband signals
- encodes navigation messages with LDPC (5G NR) and BCH(51,8)
- is validated against published test vectors and BER curves
- can cross-decode signals generated by an independent implementation
- is fully open-source (Apache 2.0) and traceable to the LSIS V1.0 spec
